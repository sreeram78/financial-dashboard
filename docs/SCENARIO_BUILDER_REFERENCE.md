## SENSITIVITY ANALYSIS INTERPRETATION

### Revenue Growth Sensitivity

**Key Insight:** Revenue is the **largest profit driver** due to operating leverage

```
Base Case Forecast: 18% Revenue Growth

If Revenue Growth = 15% (-3%)
→ Net Income impact: -20% to -30%
→ Free Cash Flow: -15% to -25%

If Revenue Growth = 21% (+3%)
→ Net Income impact: +20% to +30%
→ Free Cash Flow: +15% to +25%

Rule of Thumb: Every 1% change in revenue = 5-10% change in profit
```

**Action Items:**
- Focus on revenue drivers (market share, pricing, volume)
- Sales forecasting is critical for accuracy
- Monitor leading indicators (pipeline, win rates, churn)

---

### COGS Sensitivity

**Key Insight:** Cost of goods directly impacts gross margin (typically 55-65% of revenue)

```
Base Case: COGS = 40% of Revenue

If COGS = 38% (-2%)
→ Gross Profit: +$32M
→ Net Income: +$24M

If COGS = 42% (+2%)
→ Gross Profit: -$32M
→ Net Income: -$24M

Rule of Thumb: Every 1% COGS change = $16M net income swing
```

**Action Items:**
- Supplier negotiations critical
- Supply chain efficiency programs
- Manufacturing optimization / automation
- Pricing power to offset cost increases

---

### OpEx Sensitivity

**Key Insight:** Operating expenses grow with scale but show leverage

```
Base Case: OpEx = $240M growing at 12%

If OpEx Growth = 8% (slower)
→ Year 3 OpEx: $303M (vs. $381M base)
→ Net Income: +$39M improvement

If OpEx Growth = 16% (faster)
→ Year 3 OpEx: $459M (vs. $381M base)
→ Net Income: -$39M deterioration

Rule of Thumb: Each 1% OpEx growth change = $10M annual impact by Year 3
```

**Action Items:**
- Control headcount growth
- Monitor compensation inflation
- Implement process automation
- Optimize shared services

---

### CapEx Sensitivity

**Key Insight:** CapEx impacts cash flow but enables growth

```
Base Case: CapEx = 5% of Revenue

If CapEx = 4% (asset light)
→ Free Cash Flow: +$50M improvement
→ But may limit growth (facilities, equipment constraints)

If CapEx = 7% (heavy investment)
→ Free Cash Flow: -$100M
→ But enables 25%+ growth trajectory
→ Higher risk/reward

Rule of Thumb: 1% CapEx change = $16M annual free cash flow impact
```

**Action Items:**
- Align CapEx with growth rate assumptions
- Consider asset-light vs. owned models
- Lease vs. buy analysis
- Capacity planning

---

### Tax Rate Sensitivity

**Key Insight:** Tax rate impacts net income but limited control

```
Base Case: Tax Rate = 25%

If Tax Rate = 23% (tax benefits)
→ Net Income: +$50M improvement
→ From R&D credits, losses carried forward

If Tax Rate = 27% (higher)
→ Net Income: -$50M
→ From repatriation, state taxes

Rule of Thumb: Each 1% tax change = $8-10M net income impact
```

**Action Items:**
- Maximize R&D tax credits
- Optimize state/local tax
- Consider tax-efficient structures
- Track tax law changes

---

### Debt Paydown Sensitivity

**Key Insight:** Debt paydown improves leverage but reduces cash flexibility

```
Base Case: $10M annual debt paydown

If Debt Paydown = $5M (conservative)
→ More cash for operations/investments
→ But Debt/Equity stays elevated (higher risk)

If Debt Paydown = $20M (aggressive)
→ Rapid deleveraging
→ But limits M&A, emergency funds
→ Need strong cash generation

Rule of Thumb: Each $5M debt reduction = 0.1x D/E improvement
```

**Action Items:**
- Align with debt covenants
- Consider debt maturity profile
- Plan refinancing needs
- Monitor credit rating metrics

## FORECAST ACCURACY CHECKLIST

### Before Publishing Forecast

- [ ] **Historical Validation**: Compare to last 5 years of actuals
- [ ] **Assumption Credibility**: Align with sales pipeline, cost controls, board strategy
- [ ] **Sensitivity Review**: Understand impacts of ±1% assumption changes
- [ ] **Scenario Reasonableness**: All 3 scenarios plausible given market conditions
- [ ] **Balance Sheet Consistency**: B/S ties back to P&L (earnings retention)
- [ ] **Cash Flow Logic**: Operating CF > Net Income (healthy conversion)
- [ ] **Debt Covenant Compliance**: All years meet bank covenants
- [ ] **Peer Benchmarking**: Margins, growth aligned with comparable companies
- [ ] **External Validation**: Compare to analyst estimates if public
- [ ] **Stakeholder Review**: Finance, Operations, Sales align on assumptions

## QUARTERLY FORECAST UPDATE PROCESS

### Month 1: Establish Baseline
```
1. Load Q1 actuals
2. Update revenue to actual
3. Adjust COGS based on actual margins
4. Reforecast Q2-Q4 with new assumptions
5. Publish revised full-year forecast
```

### Month 2-3: Monitor Variance
```
1. Calculate vs. forecast variance for Q1
2. Identify key drivers of variance
3. Update assumptions for Q2 actuals
4. Track leading indicators (pipeline, bookings)
5. Adjust full-year forecast if >5% variance
```

### Every Quarter: Reforecast
```
1. Roll forward: Remove completed year, add new out-year
2. Maintain 3-year rolling horizon always
3. Update all assumptions based on new information
4. Compare to prior quarter forecast (explain changes)
5. Share variance analysis with stakeholders
```

## RED FLAGS & ADJUSTMENTS

### Red Flag: Revenue Not Meeting Plan
```
Action:
1. Analyze by customer, product, region
2. Check pipeline (is next Q better?)
3. Adjust down remaining quarters pro-rata
4. Review pricing vs. volume issue
5. Consider market share loss scenario
```

### Red Flag: Margin Compression
```
Action:
1. Increase COGS % assumption
2. Review cost of raw materials, labor
3. Assess pricing power (can you raise prices?)
4. Model cost reduction initiatives timeline
5. Evaluate product mix (low vs. high margin)
```

### Red Flag: Higher Than Expected OpEx
```
Action:
1. Identify which OpEx categories over budget
2. Separate one-time vs. recurring costs
3. Adjust OpEx growth rate upward
4. Review headcount plans (hiring ahead of revenue?)
5. Model OpEx efficiency initiatives
```

### Red Flag: Free Cash Flow Deteriorating
```
Action:
1. Check working capital assumptions
2. Review CapEx timing (spending accelerating?)
3. Assess cash conversion (AR/Inventory buildup?)
4. Model customer payment term changes
5. Plan debt refinancing if needed
```

## COMMON MISTAKES TO AVOID

❌ **Mistake 1:** Revenue growth too optimistic
   → Fix: Benchmark against market growth rate

❌ **Mistake 2:** Margin assumptions static (no leverage)
   → Fix: Model improving COGS % and OpEx as % of revenue

❌ **Mistake 3:** Tax rate set to statutory rate
   → Fix: Use effective rate and factor in credits/losses

❌ **Mistake 4:** CapEx assumption not aligned with growth
   → Fix: Higher growth = higher CapEx requirements

❌ **Mistake 5:** Debt paydown too aggressive
   → Fix: Model against free cash flow availability

❌ **Mistake 6:** Only one scenario (no risk view)
   → Fix: Always present Bear/Base/Bull scenarios

❌ **Mistake 7:** Assumptions not documented
   → Fix: Create assumption change log

❌ **Mistake 8:** No sensitivity analysis
   → Fix: Run sensitivity on top 3 variables
